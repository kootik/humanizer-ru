# Humanizer RU

Skill для Claude, который убивает AI-слоп, канцелярит и воду в русском тексте. Превращает мёртвое в живое за один проход - с голосом, ритмом и конкретикой.

Работает в Claude Code и claude.ai.

## Установка

### Claude Code

```bash
git clone https://github.com/kootik/humanizer-ru.git ~/.claude/skills/humanizer-ru
```

### Claude Code (только SKILL.md)

```bash
mkdir -p ~/.claude/skills/humanizer-ru/references
curl -o ~/.claude/skills/humanizer-ru/SKILL.md https://raw.githubusercontent.com/kootik/humanizer-ru/main/SKILL.md
curl -o ~/.claude/skills/humanizer-ru/references/patterns.md https://raw.githubusercontent.com/kootik/humanizer-ru/main/references/patterns.md
curl -o ~/.claude/skills/humanizer-ru/references/word-scanner.md https://raw.githubusercontent.com/kootik/humanizer-ru/main/references/word-scanner.md
```

### claude.ai

Скачайте `humanizer-ru.skill` из [Releases](https://github.com/kootik/humanizer-ru/releases) и загрузите через настройки проекта.

## Использование

```
/humanizer-ru
[вставьте ваш текст]
```

Или просто попросите:

```
Отредактируй этот текст: [ваш текст]
Убери воду: [ваш текст]
Сделай живым: [ваш текст]
```

### Калибровка под ваш стиль

Если нужен не общий «живой» голос, а ваш личный - приложите образец:

```
Отредактируй этот текст. Вот образец моего стиля для калибровки:
[кусок вашего старого текста]

Текст для правки:
[новый текст]
```

Скилл сначала разберёт образец (длина предложений, лексика, пунктуация, тики), потом перепишет под него.

## Три режима работы

| Режим | Что делает | Когда использовать |
|-------|-----------|-------------------|
| Полная редактура | Сканирование, переписывание, добавление голоса, верификация | По умолчанию |
| Аудит | Только диагностика - таблица проблем без изменений | «Покажи, что не так с текстом» |
| Точечная правка | Меняет только то, что просят | «Убери канцелярит, остальное не трогай» |

## 26 паттернов AI-текста

Паттерны разделены на 4 группы по степени тяжести. Скилл учитывает тип текста (маркетинг, статья, деловая переписка, техдокументация) и регулирует степень вмешательства.

Синхронизирован с upstream [blader/humanizer v2.5.1](https://github.com/blader/humanizer).

### Группа A - Критические (всегда исправлять)

| # | Паттерн | Пример |
|---|---------|--------|
| A1 | Негативные параллелизмы и хвостовые отрицания | «не просто X, а Y» / «…, без догадок» → переписать |
| A2 | Copula avoidance | «является ключевым» → «это ключевой» |
| A3 | Отглагольные существительные | «осуществлять контроль» → «контролировать» |
| A4 | Артефакты чатбота | «надеюсь, это поможет!» → убрать |
| A5 | Подобострастный тон | «отличный вопрос!» → убрать |

### Группа B - Высокие (почти всегда)

| # | Паттерн | Пример |
|---|---------|--------|
| B1 | Пассив вместо актива | «решение было принято» → «руководство решило» |
| B2 | AI-словарь | «демонстрирует», «способствует», «ключевой» |
| B3 | Цепочки родительных падежей | развитие движения за укрепление... |
| B4 | Шаблонные переходы | «важно отметить, что...» → убрать |
| B5 | Раздувание значимости | «знаменует собой ключевой этап» |
| B6 | Поверхностный анализ с деепричастиями | «символизируя...», «обеспечивая...» |
| B7 | Безличные/бессубъектные конструкции | «данные сохраняются автоматически» → «система сохраняет» |

### Группа C - Средние (по контексту)

| # | Паттерн | Пример |
|---|---------|--------|
| C1 | Рекламный язык | «захватывающий дух», «уникальный» |
| C2 | Размытые атрибуции | «по мнению экспертов» → конкретный источник |
| C3 | Правило трёх | три прилагательных подряд |
| C4 | Ложные диапазоны | «от X до Y» на разных шкалах |
| C5 | Хеджирование | «возможно, можно предположить» |
| C6 | Иностранные слова вместо русских | «имплементировать» → «внедрить» |
| C7 | Авторитетные тропы | «по сути», «главный вопрос в том» |

### Группа D - Стилистические (для голоса)

| # | Паттерн | Пример |
|---|---------|--------|
| D1 | Элегантная вариация | герой/протагонист/центральный персонаж |
| D2 | Формульные выводы | «несмотря на... сталкивается с вызовами» |
| D3 | Общие позитивные заключения | «будущее выглядит ярким» |
| D4 | Эмодзи и избыточное форматирование | эмодзи в заголовках, жирный на всём |
| D5 | Излишняя академичность | «в контексте вышеизложенного» |
| D6 | Сигнальные анонсы | «давайте разберём», «перейдём к сути» |
| D7 | Раздробленные заголовки | заголовок + строка-повтор → убрать |

## Жёсткие запреты

Эти конструкции переделываются всегда, без исключений:

- «не просто X, а Y» / «не только X, но и Y»
- длинное тире «—» (только короткое «-»)
- двоеточия после вводных слов
- риторические вопросы

## Метрики и AI-Slop Score

С v2.6.0 скилл считает **AI-Slop Score** - числовую оценку «зашумлённости» текста AI-паттернами от 0 до 100. Формула учитывает количество находок, их вес по группам и длину текста.

| Score | Состояние |
|-------|-----------|
| 0-20 | Чистый человеческий текст |
| 21-40 | Лёгкие следы AI / канцелярита |
| 41-60 | Заметный AI-слоп, рекомендуется правка |
| 61-80 | Сильный AI-слоп, требуется полная редактура |
| 81-100 | Текст почти полностью сгенерирован, нужно переписать |

В выводе скилл показывает «было N → стало M», что позволяет наглядно увидеть результат правки.

## Pre-scan через Python-скрипт

Для длинных текстов (>500 слов) можно ускорить сканирование, запустив `scripts/scan_markers.py` ПЕРЕД LLM-проходом:

```bash
# Из файла
python scripts/scan_markers.py document.txt

# Из stdin
cat document.txt | python scripts/scan_markers.py -

# Только число (для скриптов)
python scripts/scan_markers.py document.txt --quiet

# JSON-вывод (для интеграции)
python scripts/scan_markers.py document.txt --json
```

Скрипт находит маркеры по словарю всех 4 групп, считает Score и выдаёт топ-10 находок с позициями. Зависимостей нет, нужен только Python 3.8+.

Exit codes:
- `0` - score < 20 (текст чистый)
- `1` - score 20-60 (нужна редактура)
- `2` - score > 60 (сильный AI-слоп)
- `3` - ошибка ввода
