# Тест-кейсы для самопроверки

12 эталонных кейсов для верификации работы скилла. Каждый кейс — пара «До → После» + ожидаемые метрики. Используются для регрессии при изменении логики и для калибровки LLM.

## Как использовать

1. **При разработке скилла:** прогнать каждый кейс через скилл, сравнить вывод с эталоном.
2. **При оценке качества:** прогнать `scripts/scan_markers.py` на «До» и «После», сверить score.
3. **При сомнении в правке:** найти ближайший по типу кейс, использовать как образец стиля.

Кейсы покрывают 7 типов текста: маркетинг, новость, юридический, личное письмо, технический README, пост в соцсетях, пресс-релиз.

## Кейс 9. Уже хороший текст (нулевые правки)

**Тип:** статья | **Режим:** 1 | **Ожидаемый score:** <15 → не менять

**До:**
> Мой кот Мурзик не любит пылесос. Когда я включаю его, кот забивается под кровать и не вылезает минут двадцать. Я пробовал разные модели — тише не становится, кот всё равно прячется.

**После:**
> [Без изменений. Текст живой, правки минимальные.]

**Поведение скилла:** должен явно сказать «Текст живой, правки минимальные», а не выдумывать правки.

## Кейс 12. Не русский текст (отказ)

**Тип:** edge case | **Режим:** любой | **Ожидаемое поведение:** отказ

**Вход:**
> The product is a revolutionary solution that leverages cutting-edge AI to deliver unparalleled value to enterprise customers.

**Эталонный ответ скилла:**
> Этот скилл работает только с русским текстом. Для английского используй [blader/humanizer](https://github.com/blader/humanizer).

**Не должно произойти:** автоматический перевод и редактура. Скилл должен отказаться.

## Регрессионный прогон

```bash
# Сохранить каждый "До" в test_NN.txt, прогнать сканер:
for f in test_*.txt; do
  echo "$f: $(python3 scripts/scan_markers.py "$f" --quiet)"
done
```

Если score «До» отклоняется от эталона больше чем на 10 пунктов — словарь маркеров устарел или сломался.
